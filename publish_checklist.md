# Publish Checklist

Standing publishing permission was granted on 2026-06-27. Publishing is allowed when it uses an accessible public account/profile and does not cross a hard-stop boundary.

## Safe Private Prep

- [x] Product brief written.
- [x] Local CSV cleaning engine created.
- [x] Unit tests passing.
- [x] Sample input, cleaned output, and JSON report generated.
- [x] Draft API contract written.
- [x] Add hosted API wrapper or web-tool UI.
- [x] Add file-size limits and clearer error payloads.
- [x] Add benchmark samples for common messy CSV patterns.
- [x] Create marketplace description, examples, and pricing draft.
- [x] Add screenshots or terminal capture for marketplace listing.
- [x] Package for selected host or marketplace.
- [ ] Create public GitHub repository or push to accessible existing repository.
- [ ] Deploy hosted demo.

## Approval Needed

- [ ] Choose publishing platform: RapidAPI, hosted free tool, or both.
- [ ] Approve public product name and listing copy.
- [ ] Approve account/profile used to publish.
- [ ] Accept marketplace terms if needed.
- [ ] Confirm pricing.
- [ ] Confirm payout setup only if required and explicitly approved.

These items can be handled autonomously under the standing permission when the platform is already accessible and does not require spending, sensitive account changes, legal/tax/payment/payout details, OTP/MFA, CAPTCHA, or contracts. Any hard-stop item still requires exact action-time approval.

## Hard Stops

- Do not enter tax, bank, Stripe, PayPal, card, legal name/address/phone, OTP, MFA, CAPTCHA, or account-security details.
- Do not upload under the user's identity without approval.
- Do not promise support, uptime, SLA, or business availability without approval.
